# 交通事故安全診斷 — 專題需求確認

## 專題資訊
| 項目 | 內容 |
|---|---|
| 題目 | 交通事故安全診斷 |
| 資料 | 斗六市交通事故公開資料（需從網上抓取） |
| 分析 | Python 資料整理與分析：路段、時段、天候、時間、地點等風險因素 |
| 輸出 | 圖表 + 互動式地圖 + 改善建議 + ML 預測 |
| 預測目標 | 路段危險指數（Hazard Index），配合天氣預測這週風險 |
| 關鍵限制 | 建議資料必須本地端生成，**禁止呼叫外部 LLM API** |

## 設計方向（已存檔）

### 技術線
- **前端框架**：React + TypeScript + Tailwind CSS + shadcn/ui
- **動畫庫**：Framer Motion
- **圖標**：lucide-react
- **地圖**：Leaflet（互動式地圖標記路段風險）

### 頁面架構（多頁式導航）
不是單頁滾動，而是 **多個獨立頁面**，用目錄入口導航：

```
封面頁 (Landing) → 目錄頁 (GlassIcons 導航) → 各分析子頁
```

### 設計組件參考
| 組件 | 用途 | 來源 |
|---|---|---|
| **SparklesText** | 封面 Hero 大標題動畫（閃爍星星） | Magic UI |
| **GlassIcons** | 目錄頁 3D 玻璃擬態導航入口 | React Bits |
| **Features-8 (Bento Grid)** | 各分析頁的數據卡片布局 | shadcn blocks |

### 畫風參考
- 參考 `visitors.now`（godly 收錄）— 深色/玻璃擬態/Modern SaaS 風格
- 與之前 Acctual 的極簡白底風格不同，這次走深色玻璃路線

### 本地端 ML 預測策略（需實作）
1. **資料爬取**：從政府公開平台抓取斗六市事故資料
2. **特徵工程**：路段、時段、星期、天候、光線、車種等
3. **模型訓練**：scikit-learn（Random Forest / XGBoost）
4. **危險指數**：輸出 0–100 路段風險分級
5. **天氣整合**：串接公開天氣 API（氣象局 Opendata）做這週預測
6. **建議生成**：規則引擎匹配預設建議模板（非 LLM）
